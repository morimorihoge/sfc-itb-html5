CSS in Detail

In lecture #6, we have discussed the basics of CSS. In this section, we would discuss more about CSS, in detail.

More About Using Classes

In order to create CSS files in a orderly fashioned way, the basic ideas of classes must be mastered. In this section, the ideas of classes would be described along with explainations of descendant classes and multible classes definitions.

Descendant Classes

In lecture #6, we have discussed how to define specific tag names, classes, and id names as selectors. For example, say there was a class named "important" in your CSS file, as shown below. This particular class would highlight portions of text with red, and make them bold. 

    /* Highlight Important Portions */
    .important {
      font-weight: bold;
      color: red;
    }

In this particular case, this class may be applied to any tag or element. However, if you want the highlight to look different, based on the name of the tag, this could be a problem. For example, say the background color of the "header" area is black, and the background color of the "article" area is white. If you were to highlight text in each of the areas, it may be better if they had different colors.

In order to change the color, the most simplistic method would be create two different classes for each of the areas, as shown below.

    /* Highlight For Header */
    .important_header {
      font-weight: bold;
      color: blue;
    }
    /* Highlight For Article */
    .important_article {
      font-weight: bold;
      color: red;
    }

Creating individual classes may not be a problem if the number of varieties are small. However, there were a wide varieties of styles, the style sheet would become complicated with similar class names.

The solution to this problem would be use descendant selectors. By using descendant selectors, users would be able to assign the same class name for every type of tag, and obtain different designs at the same time. The following code indicates a sample of style definitions with descendant selectors. 

  /* Highlight For Header */
  header .important {
    font-weight: bold;
    color: blue;
  }

  /* Highlight For Article */
  article .important {
    font-weight: bold;
    color: red;
  }

As mentioned aboved, the same class name may be defined for any type of tag in the HTML file.

  <!-- html,headタグは省略 -->
  <body>
    <header>
      headerエリア．<span class="important">
        この部分がimportantクラスで強調されている
      </span>
    </header>
    <article>
      articleエリア．<span class="important">
        この部分がimportantクラスで強調されている．
      </span>
    </article>
    <footer>
      footerエリア．<span class="important">
        この部分がimportantクラスで強調されている.
        マッチするセレクタが無いので見た目は変わらない
      </span>
    </footer>
  </body>

Descendant selectors also enable to make things clear on CSS files. This may seem confusing at first, but it is used commonly in most HTML template. Therefore, you would need the knowledge of them upon adjusting style definitions and values in HTML templates.

Multiple Class Definitions

When defining classes as HTML attributes, you may add multiple class names to a single definition. The following code indicates an sample of multiple class definitions. When defining multiple class names, separate them with a space in between each class name. 

  /* Class For Bold Characters */
  .bold {
    font-weight: bold;
  }

  /* Class For Highlighting */
  .red {
    color: red;
  }
  
  <body>
    <article>
      <p>
        No style applied here.
      </p>
      <p class="bold">
        "Bold" class applied.
      </p>
      <p class="red">
        "Red" class applied.
      </p>
      <p class="bold red">
        Both "bold" and "red" classes applied.
      </p>
    </article>
  </body>

Open the sample code with a browser

Multiple class definitions are useful upon avoiding duplicate definitions and making style sheets simpler. Therefore, we recommend you use them upon making your own style sheet.
